2. 产品概述
2.1 产品定位
设计师内部SaaS系统是在现有"全景渲染系统SaaS后台V2.0"基础上扩展的设计师案例复用赋能系统。系统以"案例结构化沉淀 → 智能检索 → 一键复用 → NAS安全隔离 → 设计软件无缝衔接"为核心链路，解决室内设计师在选品、风格搭配、3D模型获取、案例跨人复用等环节的效率瓶颈。
核心定位：不替代设计师的创意工作，而是将已验证的设计经验（产品搭配方案+3D模型组合）系统化沉淀并高效复用，让设计师从"重复造轮子"走向"站在前人肩膀上"。

2.2 核心价值主张
维度 | 价值主张 | 量化目标
效率提升 | 将设计师选品环节从2-4小时压缩至30分钟以内，设计交付周期缩短30%+ | 选品效率指数 ≤30min/次
质量保障 | 通过结构化案例参考和已验证搭配方案复用，降低新手搭配失误率，统一团队设计质量 | 案例复用渗透率 ≥40%
知识沉淀 | 将设计师个人经验转化为组织级数字资产，实现部门/站点级设计知识的规模化复用 | 跨设计师复用率 ≥30%

2.3 目标用户
用户类型 | 代表画像 | 核心诉求 | 核心使用场景
初级设计师 | 林悦（25岁，入行1年） | 快速出图、风格不出错、选品有据可依 | 浏览案例库 → 一键复用搭配方案 → 微调 → 出图
资深设计师/部门主管 | 陈磊（35岁，入行10年） | 案例沉淀、团队复用、质量统一 | 标记推荐案例 → 管理风格标签 → 查看团队复用数据
自由设计师 | 王明远（30岁，独立执业3年） | 跨项目高效复用、快速获取模型 | 个人案例沉淀 → 相似项目一键复用 → 局部替换

2.4 与现有系统关系
本系统是全景渲染系统SaaS后台V2.0的功能扩展版本（V3.0），遵循"扩展而非替代"原则：
关系类型 | 说明
复用 | 完全复用现有全景合成、全景编辑器（18+功能）、作品管理、用户体系、多租户架构、数据看板基础框架
扩展 | 扩展NAS存储能力（从全景图存储扩展为全景图+3D模型+设计源文件+产品图统一存储）、扩展作品管理（增加"案例"维度）、扩展权限体系

3.5 多租户数据隔离策略
隔离层级 | 策略 | 技术实现
数据库 | Shared Database Separate Schema | 站点级Schema隔离
应用层 | 站点ID + 部门ID 双重过滤 | 所有查询自动注入租户过滤条件

3.6 技术架构简述
新增微服务：
服务名 | 职责 | 关键技术
case-service | 案例CRUD、案例检索、风格标签管理、复用记录 | MySQL + Elasticsearch
product-service | SKU管理、3D模型管理、分类标签、外部API对接 | MySQL + OSS
nas-service | NAS文件操作、副本创建、配额管理、存储监控 | NAS API + MySQL
conversion-service | 3D模型格式转换、CAD图纸解析 | 异步任务 + 临时存储

NAS同步机制：
1. 复用触发副本创建：通过消息队列异步执行，前端轮询任务状态
2. 副本创建原子性：先完整复制 → 校验文件一致性 → 更新数据库状态 → 通知用户
3. 失败回滚：副本创建失败时，清理已复制文件，重置数据库状态，通知用户重试
4. 定期一致性校验：每日凌晨对NAS文件与数据库记录做一致性校验

4.3 权限继承与覆盖规则
1. 层级继承：上层角色自动拥有下层角色的所有权限
2. 站点管理员继承部门管理员权限，但仅限本站点
3. 超级管理员拥有所有权限，无任何限制
4. 权限覆盖：当用户拥有多个角色时，取权限并集
5. 数据级权限：通过数据过滤实现（如部门管理员只能看到本部门数据）

6.2 关键交互流程
1. 案例创建流程：
   设计师上传3D源文件 → 系统自动提取产品SKU → 填写案例信息 → 打标签 → 保存到个人案例库

2. 案例复用流程：
   浏览案例库 → 选择目标案例 → 点击"一键复用" → 系统创建NAS副本 → 生成个人案例副本 → 可编辑修改

3. 产品搜索与下载：
   搜索产品 → 查看产品详情 → 选择3D模型格式 → 点击下载 → 后台转换（如需） → 下载到本地

4. 数据统计查看：
   进入数据看板 → 选择时间范围 → 查看各项指标 → 导出报告
