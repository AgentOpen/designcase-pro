- 卡片：显示「设计软件：🟥 3DMax」单一标识；封面角标、空间瓦片同步单一。
- 详情：标题改为「设计软件」，说明「本案例由 X 设计，源文件为 .xxx 格式，复用下载后需用 X 打开」；3D模型文件也只给该软件源文件 + 通用 FBX 导出。
- 我的案例表单：软件改为单选（radio）。对比弹窗「设计软件」行取单一值。NAS 每个项目单一设计软件。
- 软件筛选兼容单一字段。

## 本次更新（角色差异化加深）
在上一版 RBAC 基础上补齐三处身份差异：
- 个人中心「系统管理」（用户管理/权限配置/审计日志）仅组长可见，设计师该 Tab 隐藏；钱包「组长工具」也仅组长显示。
- 「我的案例」按当前身份过滤：每位设计师有自己的项目种子，只看/保存属于自己的项目，新建案例归属当前身份（编号/作者随之变化），复制来的项目也归当前人。
- 设计师榜单高亮当前登录身份（演示为林悦时高亮林悦），组长/设计师都能看榜（透明激励）。

## 本次更新（角色权限差异化 RBAC + 复制整个项目）
设计了统一权限体系 GDRole.can(action)，组长（既是设计师也是管理者，权限是设计师的超集）与设计师看到的功能/数据不同，随左侧身份切换实时变化：

组长（管理端）专属：新建协作项目、给任意人分配任务、编辑任意项目进度/状态、删除项目、登记/分配外部派单、修改任意成员状态、发放积分、审核案例积分、查看团队数据总览、邀请同事；仪表盘显示团队视角文案 + 待审核提醒。

设计师（用户端）：
- 项目协作只看自己参与的项目，进度只读（靠完成自己任务推进），无新建/分配/邀请按钮；
- 任务看板强制只看自己的任务、只能推进自己的任务、无添加/删除他人任务；
- 外部派单只看派给自己的；成员状态只能改自己；
- 仪表盘隐藏"团队数据"Tab，显示个人视角；
- 仍可复用案例、复制整个项目、产出自己的案例赚积分。

新增「复制整个项目」：灵感库 / NAS / 团队项目卡与项目详情页均可一键把整个项目（含全部空间）复制到「我的案例」，并按空间数进行积分结算转给原作者（区别于仅复用单个空间）。
